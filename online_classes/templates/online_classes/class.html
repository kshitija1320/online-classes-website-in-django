    • Requirement Gathering And Analysis

      We will take requirements of project from MNC and divide this project in different modules. We will hand over one module for one student. Student will be work on the analysis of his module like what algorithms and database tables will be use in his module.

      Specification and Design

      After the analysis, student will be prepare a specification document which contain all the analysis regarding his module. Student will be present this document in the status meeting. We will also help to choose correct database design and algorithms before the actual implementation start.

      Implementation / Coding

      After the specification document present, student will be work on actual implementation. Student must show progress of the module in status meetings and also discuss on the implementation if he/she face issue.

      Testing

      It is self-testing, cross-testing and unit testcase creation phase. After the self-testing of his module, It will be cross test by other student. Each student necessary to create the unit testcase of his module after the implementation. Code and unit testcase will be submit together.

      Final code deployment

      We will join each module and create a final project. Now all modules will be run together. If any module does not give expected output then student will be correct that module only.
